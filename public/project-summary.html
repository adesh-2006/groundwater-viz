🏗️ Architecture Overview

Tech Stack: Next.js 16 (Turbopack) • React 19 • Framer Motion • Three.js / R3F • Leaflet • GSAP
Data Sources: India WRIS ArcGIS (arc.indiawris.gov.in) • India WRIS Dataset API (indiawris.gov.in) • CGWB GWR-2024
Storage: In-memory (globalThis singleton via sensorStore.js) — no external database
Caching: Server-side in-memory Maps with 5–30 min TTL per route • Max 50 entries LRU

⚡ API Routes (Backend)

🔐 Authentication

File Method Endpoint Role & Description
api/auth/login/route.js POST /api/auth/login Login. Checks email/password against in-memory user list. Returns base64 token + user object with role. Demo: admin@aquaviz.com/admin123.
api/auth/signup/route.js POST /api/auth/signup Registration. Creates new user with auto-incremented ID. Validates password ≥ 6 chars. Returns token for auto-login.

📊 Sensor / Dashboard Data

File Method Endpoint Role
api/groundwater/route.js GET /api/groundwater Dashboard Summary. Reads from sensorStore (getAllSensors + getStats). Enriches with real India WRIS rainfall data. Returns: active_sensors, avg_water_level, avg_ph, rainfall_today, recharge rate.
api/rainfall/route.js GET /api/rainfall Rainfall Data. Simulated hourly/monthly rainfall generator. 6 Indian regions with status. Types: summary, hourly (24h), monthly (12 months).
api/sensor-history/route.js GET /api/sensor-history Sensor History + Real WRIS. Combines local pushed readings (sensorStore) with real India WRIS groundwater level timeseries. Fetches water quality from ArcGIS. Maps sensor states to WRIS locations. 316 lines.

🛠️ Admin CRUD

File Methods Endpoint Role
api/admin/sensors/route.js GET POST PUT DELETE /api/admin/sensors Sensor CRUD. List all, add new, update, delete, push readings. All operations go through sensorStore. POST action="push-reading" updates sensor readings.
api/admin/sensors/history/route.js GET /api/admin/sensors/history Sensor Reading History. Returns historical readings for a specific sensor_id from sensorStore (limit parameter, default 50).
api/admin/users/route.js GET /api/admin/users User List. Returns 4 hardcoded users (admin, operator1, field_eng, analyst).
api/admin/stats/route.js GET /api/admin/stats Admin Statistics. Returns totalUsers, totalSensors, activeSensors, criticalReadings from sensorStore.

🌐 India WRIS Integration

File Method Endpoint Role
api/wris/route.js GET /api/wris WRIS Dataset Proxy. Proxies 7 India WRIS datasets: groundwater-level, rainfall, river-water-level, river-discharge, temperature, humidity, reservoir. 5-min cache TTL.
api/wris/realtime/route.js GET /api/wris/realtime 🔴 Live Monitor Backend. 477 lines. 3 actions: overview (GWR-2024 national stats + block categories + station counts), water-level (live readings from 10 preset locations), water-quality (station chemical params). SSL bypass for ArcGIS.
api/wris/arcgis/route.js GET /api/wris/arcgis ArcGIS CORS Proxy. 274 lines. Proxies India WRIS ArcGIS MapServer queries. Handles station listing, state/district filtering, statistics queries, water quality data. SSL bypass + retry logic. 10-min cache.
api/wris/master/route.js GET /api/wris/master Master Data. Fetches dropdown lists: states, districts (by state), agencies (by state+district), basins, stations. 30-min cache for stable reference data.

📦 Shared Library

File Lines Role
lib/sensorStore.js 205 In-Memory Sensor Data Store. Uses globalThis singleton to share state across all API routes (prevents Turbopack module duplication). Manages 12 default DWLR sensors with auto-generated 30-day history. Exports: getAllSensors, getSensorById, addSensor, updateSensor, pushReading, getHistory, deleteSensor, getStats. Computes quality status (Good/Moderate/Critical) based on water level & pH.
